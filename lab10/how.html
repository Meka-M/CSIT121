How Subresource Integrity is Used

You will use SRI by starting with a cryptographic hash that is base64-encoded. While doing so, you will be using the integrity attribute, a link element, or script element, that will be used along with rel=“stylesheet” or something else like rel=“preload.” The integrity value will have a prefix before the hash. Also, the use of the crossorigin attribute will lead to the prevention of cross-origin attacks. An example would look like: sha256-2cf24dba5fb0a30e26e83b2ac5b9e29e1b161e5c1fa7425e73043362938b9824.

There are plenty of hash generators that you can find online also to make one that is linked to a URL. So, when looking at the script element, you want to add the src attribute first to place the script. Then, you will place that hash with the prefix within the integrity attribute. Finally, you will end it with a cross-origin attribute and a closed script tag. An example would be: opening script src=https://website.com/website-framework.js integrity=sha256- 2cf24dba5fb0a30e26e83b2ac5b9e29e1b161e5c1fa7425e73043362938b9824 crossorigin=anonymous closing script.

As for the link element, you are focusing on the stylesheet. So, with this one, you want to start with the link element. Within it, you will use the rel attribute and also place the link. The integrity attribute is also within that element, and you will wrap it up with the crossorigin attribute. The example for this one would be: “ opening link rel=stylesheet href=https://website.com/style.css integrity=sha256- 2cf24dba5fb0a30e26e83b2ac5b9e29e1b161e5c1fa7425e73043362938b9824 crossorigin=anonymous closing tag.
